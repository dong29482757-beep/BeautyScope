# BeautyScope

쿠팡, 무신사, 올리브영에서 모은 화장품 리뷰 538,774건을 분석해서 보여주는 웹 서비스입니다.
"이 제품 진짜 괜찮은가?"를 리뷰 몇 개 읽고 판단하지 말고, 데이터로 보자는 생각에서
시작했습니다.

원래는 Python(pandas, scikit-learn)으로 감성분석 결과를 Streamlit으로 보여주는
프로젝트였는데, 학교 과제 형식에 맞춰 Spring MVC + MyBatis + Oracle + JSP로 다시
짰습니다. 그 과정에서 회원가입/로그인, 관리자 페이지, 소셜 로그인, 얼굴 인식 로그인까지
하나씩 붙여나갔습니다.

## 뭘 할 수 있나

- 상품을 카테고리로 걸러보고, 평점/최신순으로 정렬하고, 검색할 수 있습니다
- 상품 상세에서 리뷰를 비동기로 불러오고, 평점 분포를 차트로 봅니다
- 리뷰를 8개 속성(보습/수분, 트러블/자극, 발림성/흡수력 등)으로 나눠서 분석해두고,
  장점/단점으로 나눠서 보여줍니다
- 카테고리별 랭킹과 브랜드 비교 차트가 있습니다
- "AI 추천"이라는 이름으로, 신경 쓰는 속성을 고르면 점수를 매겨 추천해줍니다
  (자세한 내용은 아래에 따로 적었습니다 — 이름은 AI지만 머신러닝은 아닙니다)
- 회원가입/로그인, 관리자 페이지(회원/상품/리뷰 관리)
- 카카오·네이버 소셜 로그인
- 얼굴로 로그인하는 기능도 있습니다 (역시 아래에 어떻게 만들었는지 적어뒀습니다)
- 상품 사진도 다 붙여놨습니다 (2,393개 중 2,370개, 약 98.5%)

## AI 추천, 사실대로 말하면

이름은 "AI 추천"이지만 학습된 모델은 아닙니다. 데이터를 분석한 결과를 가지고
**점수를 매기는 공식**입니다.

```
점수 = 선택한 속성의 만족도(50%) + 전체 리뷰 만족도(30%) + 평점(20%)
```

여기서 신경 쓴 부분은, 리뷰가 5개뿐인 제품이 우연히 다 긍정이라고 1위로 올라가면
안 되니까 "베이지안 평균"이라는 방법으로 보정을 했습니다. 리뷰가 적으면 점수를
카테고리 평균 쪽으로 끌어당기는 식입니다. 가중치 50/30/20은 제가 정한 값이고,
회귀분석 같은 걸로 데이터에서 자동으로 찾아낸 값은 아닙니다. (화장품 리뷰는 평점이
대부분 4~5점에 몰려있어서, 회귀로 가중치를 학습시켜도 신뢰할 만한 결과가 나오기
어렵다고 판단했습니다.)

## 얼굴 인식 로그인, 어떻게 만들었나

처음에는 face-api.js라는 라이브러리가 제공하는 사전학습된 얼굴인식 모델을 그대로
썼는데, "그건 우리가 만든 게 아니지 않냐"는 얘기가 나와서 인식 알고리즘 자체를
직접 구현했습니다.

- **얼굴이 어디 있는지 찾는 것**: face-api.js의 TinyFaceDetector를 그대로 씁니다.
  이건 "위치 찾기"일 뿐이라 직접 만들 이유가 크지 않다고 판단했습니다.
- **이 얼굴이 누구인지 구분하는 것**: 직접 만들었습니다. **고유얼굴(Eigenfaces)**이라는
  꽤 오래된(1991년) 방법인데, 원리는 이렇습니다:
  1. 얼굴 영역을 48×48 흑백 픽셀로 줄여서 숫자 2,304개짜리 벡터로 만듭니다
  2. 등록된 모든 얼굴 벡터로 평균 얼굴을 구하고, 행렬을 하나 만듭니다
  3. 이 행렬을 고유분해해야 하는데, 자바에 마땅한 라이브러리가 없어서
     **야코비 회전법**이라는 고전적인 수치해석 알고리즘을 직접 짰습니다
     (`EigenfaceRecognizer.java`)
  4. 그렇게 나온 고유벡터들로 "얼굴 공간"을 만들고, 새로 들어온 얼굴을 그 공간에
     투영해서 거리를 재서 누구와 가장 가까운지 비교합니다

직접 테스트해보니 같은 사람(같은 그룹) 얼굴끼리는 거리가 12~22 정도로 가깝게
나오고, 다른 사람과는 1045~1054로 뚜렷하게 멀어지는 걸 확인했습니다.

다만 한계는 있습니다. 등록된 사람이 1명뿐이면 고유얼굴 자체를 만들 수가 없어서
그럴 땐 원본 픽셀끼리 그냥 거리 비교로 넘어갑니다. 그리고 조명이나 얼굴 각도가
바뀌면 꽤 민감하게 흔들립니다. 그래서 이건 데모용이고, 진짜 보안이 필요한 곳에
쓸 수준은 아닙니다.

## 기술 스택

- Spring MVC 6 + MyBatis, Java 17
- Oracle (XE)
- JSP/JSTL, Chart.js는 CDN으로
- 로그인은 Spring Security 없이 세션으로 직접 처리, 소셜 로그인은 카카오/네이버 OAuth2

## 폴더 구조

```
kr.ac.kopo
├── product   상품 목록/검색/상세/랭킹
├── review    리뷰 조회, 관리자 리뷰 등록
├── aspect    속성 분석 + AI 추천
├── member    회원가입/로그인/얼굴인식/소셜로그인
├── admin     관리자 페이지
└── common    공통 인터셉터, 예외 처리
```

## 데이터는 어디서 왔나

쿠팡/무신사/올리브영을 Python으로 크롤링해서 리뷰 538,774건, 상품 2,393개를
모았습니다. Okt로 형태소분석을 하고 속성별 감성분석(ABSA)을 돌린 결과를 Oracle에
넣어서 Spring이 그걸 조회/통계/추천에 쓰는 구조입니다.

상품 사진은 무신사·올리브영은 직접 크롤링했는데, 쿠팡은 자체적인 봇 차단이 있어서
그걸 우회하는 대신 네이버 쇼핑 검색으로 같은 상품명을 검색해서 대표 이미지를
가져왔습니다 (정직하게 말하면 이래서 가끔 진짜 그 상품이 아닌 비슷한 제품 사진이
붙는 경우도 있습니다).

## DB 테이블 간단히

- `T_PRODUCT` : 상품 정보 (플랫폼+상품ID가 PK, 평균평점, 리뷰수, 이미지 경로 등)
- `T_REVIEW` : 리뷰 538,774건
- `T_ASPECT_SENTIMENT` : 상품×속성별 감성 분석 결과
- `T_MEMBER` : 회원 정보 (비밀번호는 해시로 저장, 얼굴 벡터 포함)

## 실행하려면

1. `sql/` 폴더 안의 스키마 파일들을 Oracle에 순서대로 실행
   (`beautyscope_schema.sql` → `member_schema.sql` → `member_face_schema.sql` → `aspect_schema.sql`)
2. `src/main/resources/config/spring/spring-mvc.xml`에서 DB 접속 정보 바꿔주기
3. 소셜 로그인 쓰려면 `oauth.properties.example`을 `oauth.properties`로 복사해서
   카카오/네이버 키 채우기 (이 파일은 git에 안 올라갑니다)
4. `mvn clean package` 빌드해서 Tomcat에 배포

## 솔직히 말하는 한계

- "AI 추천"은 학습된 모델이 아니라 통계 공식입니다
- 얼굴 인식은 데모 수준이고, 보안 인증으로 쓸 정확도는 아닙니다
- 상품 이미지는 전체의 약 1%가 없습니다 (크롤링 당시 페이지가 없었거나 검색에서 못 찾음)
- 상품 이미지 폴더(800MB 넘음)는 용량 문제로 이 저장소에는 안 올렸습니다
