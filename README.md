# BeautyScope

쿠팡 · 무신사 · 올리브영 화장품 리뷰 538,774건을 분석해서, 카테고리/속성별 만족도와 추천을
보여주는 Spring MVC 기반 웹 서비스입니다.

원래는 Python(pandas/scikit-learn/Streamlit)으로 만든 리뷰 감성분석 프로젝트였고, 이를
Spring MVC + MyBatis + Oracle + JSP로 다시 구현하면서 회원/관리자/소셜 로그인/얼굴 인식
로그인까지 확장했습니다.

## 주요 기능

| 기능 | 설명 |
|---|---|
| 상품 목록/검색/상세 | 카테고리 필터, 평점/최신 정렬, 페이징, 상품명/리뷰 키워드 검색 |
| 리뷰 보기 | 상품 상세에서 비동기(fetch) 로딩, 평점 분포 차트(Chart.js) |
| 속성 기반 분석(ABSA) | 8개 속성(보습/트러블/지속력/발림성 등)별 긍정 비율을 장점/단점으로 표시 |
| 카테고리 랭킹 | 카테고리별 평점 랭킹, 브랜드 비교 차트 |
| AI 추천 | 카테고리 + 속성(복수 선택) 기반 합성 점수 추천 — 아래 [AI 추천 로직](#ai-추천-로직) 참고 |
| 회원/관리자 | 일반 회원가입·로그인, 관리자 회원/상품/리뷰 관리(추가/수정/삭제) |
| 소셜 로그인 | 카카오·네이버 OAuth2 |
| 얼굴 인식 로그인 | 얼굴 검출만 face-api.js, 인식(매칭)은 고유얼굴(Eigenfaces/PCA)을 직접 구현 — 아래 [얼굴 인식 로직](#얼굴-인식-로직) 참고 |
| 상품 이미지 | 무신사·올리브영은 직접 크롤링, 쿠팡은 네이버 쇼핑 검색을 통해 대표 이미지 매칭 |

## AI 추천 로직

머신러닝 모델(학습된 가중치)이 아니라, 우리가 이미 분석해 둔 리뷰 통계를 가중합하는
**통계 기반 추천 로직**입니다. 솔직하게 명시합니다:

```
점수 = 0.5 × (선택한 속성들의 만족도, 베이지안 보정)
     + 0.3 × (전체 리뷰 만족도, 베이지안 보정)
     + 0.2 × (평점 / 5 × 100)
```

리뷰 수가 적은 제품이 우연히 높은 점수로 노출되는 것을 막기 위해, 만족도는
[베이지안 평균](https://en.wikipedia.org/wiki/Bayesian_average)으로 보정합니다
(`(긍정수 + k × 카테고리평균) / (전체수 + k)`). 가중치(0.5/0.3/0.2)는 사람이 정한
값이며, 데이터로부터 학습된 값이 아닙니다 — 자세한 한계는
`AspectServiceImpl.java`의 주석을 참고하세요.

## 얼굴 인식 로직

사전학습된 얼굴인식 신경망(임베딩 모델)을 그대로 가져다 쓰지 않고, **고유얼굴
(Eigenfaces, PCA 기반 얼굴인식, Turk & Pentland 1991)** 알고리즘을 외부 선형대수
라이브러리 없이 직접 구현했습니다 (`EigenfaceRecognizer.java`).

- **얼굴 검출(어디 있는지)**: face-api.js의 TinyFaceDetector만 사용 — 위치만 찾음
- **얼굴 인식(누구인지)**: 직접 구현
  1. 검출된 영역을 48×48 흑백 픽셀(2,304차원 벡터)로 변환해 서버로 전송
  2. 등록된 모든 얼굴 벡터로 평균 얼굴 + 그램행렬(N×N)을 계산
  3. **야코비(Jacobi) 회전법**으로 그램행렬을 직접 고유분해 (외부 라이브러리 미사용)
  4. 고유벡터로 고유얼굴 기저를 만들고, 새 얼굴을 그 공간에 투영해 유클리드 거리로 비교

표본(N)이 픽셀차원(D)보다 훨씬 작을 때 D×D 공분산행렬 대신 N×N 그램행렬을
고유분해하는 고전적인 트릭을 썼습니다. 등록자가 1명뿐이면 PCA가 의미 없으므로
원본 픽셀 벡터 거리로 자동 대체됩니다.

정확도는 상업용 얼굴인식 API보다 낮고(조명/각도 변화에 민감), 임계값도 작은
데모 데이터로 추정한 값이라 **데모 수준이며 보안 등급 인증으로는 쓸 수 없습니다.**

## 기술 스택

- **Backend**: Spring MVC 6, MyBatis, Java 17
- **DB**: Oracle (XE)
- **View**: JSP + JSTL, Chart.js (CDN)
- **인증**: 세션 기반 로그인(Spring Security 미사용), 카카오/네이버 OAuth2, 얼굴 인식(검출은 face-api.js, 인식은 직접 구현한 Eigenfaces)
- **빌드**: Maven

## 패키지 구조

```
kr.ac.kopo
├── product   상품 목록/검색/상세/랭킹
├── review    리뷰 비동기 조회/관리자 등록
├── aspect    속성기반 분석 + AI 추천
├── member    회원가입/로그인/로그아웃/얼굴인식/소셜로그인
├── admin     관리자: 회원/상품/리뷰 관리
└── common    공통 인터셉터(관리자 접근 제어), 전역 예외 처리
```

## 데이터 파이프라인

```
쿠팡/무신사/올리브영 크롤링 (Python)
  → 리뷰 538,774건 + 상품 2,393개 CSV
  → Okt 형태소분석 + 속성기반 감성분석(ABSA)
  → Oracle 적재 (T_PRODUCT, T_REVIEW, T_ASPECT_SENTIMENT)
  → Spring MVC가 조회/통계/추천 로직 수행
```

상품 이미지는 무신사·올리브영은 상품 상세페이지의 `og:image`를 직접 크롤링했고,
쿠팡은 자체 사이트의 봇 차단 정책을 우회하지 않는 대신, 네이버 쇼핑 검색에서 같은
상품명으로 대표 이미지를 가져왔습니다 (전체 2,393개 중 2,370개, 약 98.5% 수집).

## DB 스키마 (요약)

- `T_PRODUCT` — 플랫폼/상품ID(복합 PK), 상품명, 브랜드, 카테고리, 리뷰수, 평균평점, 긍/부정 리뷰수, 이미지 경로
- `T_REVIEW` — 리뷰 538,774건 (평점, 감성, 본문, 작성일, 닉네임)
- `T_ASPECT_SENTIMENT` — 상품×속성×감성별 리뷰 개수 (ABSA 결과)
- `T_MEMBER` — 회원 정보(비밀번호 해시, 가입수단, 얼굴 디스크립터)

## 실행 방법

1. Oracle에 `sql/` 폴더의 스키마(`beautyscope_schema.sql`, `member_schema.sql`,
   `member_face_schema.sql`, `aspect_schema.sql`) 실행
2. `src/main/resources/config/spring/spring-mvc.xml`의 `dataSource` 빈에서
   DB 접속 정보(url/username/password) 수정
3. 소셜 로그인을 쓰려면 `src/main/resources/config/oauth.properties.example`을
   `oauth.properties`로 복사해서 카카오/네이버 Client ID·Secret 입력
4. `mvn clean package`로 빌드 후 Tomcat에 배포

## 한계 및 알려진 이슈

- AI 추천은 통계 공식이며 학습된 모델이 아닙니다 (위 [AI 추천 로직](#ai-추천-로직) 참고)
- 얼굴 인식 로그인은 데모 수준이며 보안 등급 인증으로 사용할 수 없습니다 (자세한 내용은 [얼굴 인식 로직](#얼굴-인식-로직) 참고)
- 일부 상품(전체의 약 1%)은 이미지가 없습니다 (크롤링 시점에 페이지가 없거나 검색 매칭 실패)
